## 종료 원칙

- 완료하지 않은 목표를 완료로 표시하지 않는다.
- `.omo/ulw-loop/goals.json`과 `ledger.jsonl`은 당시 판단과 증거를 보존하는 읽기 전용 이력으로 남긴다.
- 일반 작업자는 `ulw-loop complete-goals`를 실행하지 않는다.
- ULW를 다시 시작하려면 기존 장부를 자동 재개하지 말고, 현재 `main`과 열린 PR을 먼저 대조한 뒤 명시적으로 재개한다.
- 모든 실제 구현은 최신 `main`에서 기능별 브랜치와 별도 PR로 진행한다.

## 일반 작업 우선순위

1. PR #48을 검수해 정본과 중단 작업 목록을 `main`에 반영한다.
2. PR #49를 검수·머지해 잘못 합쳐진 계주 선수명과 사람 이름 형태의 종목이 공개 색인에 나오지 않게 한다.
3. PR #50에서 PostgreSQL 권리 요청 저장을 TDD로 구현한다.
4. 인증·미성년·관리자 보안(G005)을 별도 PR로 진행한다.
5. CI·백업·복구·저장소 비공개 전환(G006)을 완료한다.
6. 원본과 승인 데이터를 비공개 저장소로 옮긴 뒤 수집 파이프라인(G007)을 통합한다.
7. 마지막으로 기록 중심 UX와 커뮤니티 공개 조건(G008)을 검증한다.

의존성을 무시해 G007 역사 데이터 승격이나 G008 커뮤니티 공개를 먼저 진행하지 않는다.
