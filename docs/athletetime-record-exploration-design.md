# AthleteTime — 기록 탐색·비교 기능 설계

## 0. 이 문서가 답하는 것

선생님 지시를 기능 단위로 분해한다:
1. 내 기록뿐 아니라 **다른 사람의 기록 / 한국신기록 / 부별 기록 / 3년치 기록 / 선수별 추이 비교**도 탐색 가능하게
2. **다종목 선수**(장거리: 1500·3000mSC·5000·10000·하프·풀코스·계주 등 5종목 이상)의
   종목별 기록 추이를 **검색·비교**에 쓸 수 있게

> ⚠️ 신뢰 전제(불변): AthleteTime은 **공식 랭킹/기록 기관이 아니다.** "모은 공개 기록"의 검색·해석 도구다.
> 모든 비교·추이·신기록 표시는 "AthleteTime이 모은 범위 안에서"임을 고지한다.

### 1.1 현재 구조의 핵심 갭

- 선수 프로필 `events[]`에 **종목별 `{eventKey, eventLabel, recordCount, best}` 이미 존재**. (다종목 데이터는 있다)
- 그러나 `recordTrail`/`summary.delta`는 **best 종목 1개에만** 묶여 있다.
  → 다종목 선수의 "종목을 바꿔가며 추이 보기"가 **불가능**. ← 이번 작업의 핵심.
- "한국신기록" 플래그(`isNationalRecord` 등)는 인덱스에 **없음**. 별도 판단/표시 설계 필요.
- 선수 vs 선수 / 시즌 vs 시즌 **비교 화면 자체가 없음**.

## 2. 탐색 축 (Exploration Axes) 정의

사용자가 기록을 보는 "관점"을 5축으로 정리한다. 기존 화면을 크게 갈아엎지 않고 **축을 더하는** 방향.

| 축 | 질문 | 현재 | 추가 필요 |
|---|---|---|---|
| **A. 선수별** | "이 선수의 기록은?" | ✅ athlete profile | 종목 전환·종목별 추이(2.1) |
| **B. 종목·시즌별** | "2024 남고 1500m 순위는?" | ✅ season-records | 부문(division) 전환 노출 강화 |
| **C. 기록 추이** | "이 선수 3년간 어떻게 변했나?" | △ best 1종목만 | 종목별·기간 선택 추이(2.1) |
| **D. 비교** | "두 선수/두 시즌 비교" | ❌ | 비교 화면(2.2) |
| **E. 신기록/하이라이트** | "한국신·대회신·시즌 베스트" | ❌ | 신기록 축(2.3, 보수적) |

### 2.1 다종목 선수 — 종목별 추이 (핵심)

**문제:** 장거리 선수는 1500·3000mSC·5000·10000·하프·풀코스·계주를 오간다.
한 종목만 추이로 보면 선수의 실제 활동을 못 본다.

**설계:**
- 선수 프로필에 **종목 선택 탭/칩**을 둔다 (events[] 기반).
  - 1종목 선수에겐 탭을 숨긴다 (58%는 불필요).
  - 다종목 선수는 종목 칩을 누르면 그 종목의 추이·베스트·delta가 갱신.
- 추이는 **종목별로 분리**해서 그린다. 서로 다른 단위(초 vs 거리 vs 점수)를 한 그래프에 섞지 않는다.
  - 트랙(시간↓ 좋음)·필드(거리/높이↑ 좋음)·종합(점수↑) 방향이 다르므로 **종목 단위로만** 추이.
- 기간 선택: "전체 / 최근 3년 / 최근 1년" (실측상 3년치 비교 수요 충분).

> 신뢰 가드: 종목별 비교 기록 2개 미만이면 추이 대신 "비교할 기록이 아직 적어요" 안내.
> "향상/하락" 같은 평가 단어 금지 → "+0.32" 같은 **사실 변화량**만, 색은 중립.

### 2.2 비교 (Compare) — 선수 vs 선수 / 시즌 vs 시즌

**옵션 1 (1차):** **같은 종목 안에서** 두 선수의 기록을 나란히.
  - 예: 2024 남고 5000m에서 A선수 vs B선수의 시즌 베스트·기록 흐름.
  - 단위가 같은 종목 내에서만 비교 (단위 혼동 방지).
**옵션 2 (후순위):** 한 선수의 **시즌 간** 비교 (2023 vs 2024 같은 종목 베스트).

> 신뢰 가드: 비교는 "누가 더 우월"이 아니라 "모은 기록을 나란히 본다". 승자 강조·랭킹 단정 금지.
> 동명이인 가능성 고지 유지. 출처·대회·풍속(legal 여부) 함께 표기.

### 2.3 신기록/하이라이트 축 — **가장 보수적으로**

"한국신기록"은 공식 인증의 영역이라 우리가 단정하면 위험하다.

- ❌ "한국신기록" 단정 표시 금지 (공식 인증 데이터 없이는).
- ✅ 대신 **"AthleteTime이 모은 기록 중 최고"** / **"이 시즌 모은 기록 중 1위"** 처럼
  **범위를 명시한** 표현만 사용. (forbidden words 원칙 §과 일치)
- 공식 신기록 데이터(공공데이터/연맹)가 검증 연동되기 전까지 "신기록" 단어는 쓰지 않는다.

## 3. API 설계 (Codex 도메인 — 제안)

기존 엔드포인트는 유지하고 **확장**한다. 모두 읽기전용, person_no 미사용.

### 3.1 athlete profile 확장 — 종목별 추이
현재 `GET /analytics/athletes/:athleteKey`의 응답에 추가:
```
events[]: { eventKey, eventLabel, recordCount, best,
            trail: TrailPoint[],          // ← 종목별 추이 (NEW)
            delta: { from, to, display, improved } | null,  // ← 종목별 변화량 (NEW)
            comparableCount }             // 비교 가능한(같은 종목·단위) 기록 수
```
- 기존 최상위 `recordTrail`/`summary.delta`는 **하위호환 위해 유지**(best 종목).
- TrailPoint는 기존 `toTrailPoint` 재사용.

### 3.2 기간 필터 (선택)
`GET /analytics/athletes/:athleteKey?since=2023` → trail/delta를 해당 연도 이후로 한정.
(프론트에서 클라이언트 필터로도 가능하므로 우선순위 낮음. 서버 지원 시 정확.)

### 3.3 비교 API
```
GET /analytics/compare?eventKey=&divisionKey=&season=&athleteKeys=key1,key2
→ { eventKey, eventLabel, season, divisionLabel,
    athletes: [{ athleteKey, name, team, ambiguity, trail[], best, seasonBest }],
    disclaimer }
```
- 같은 종목·시즌 컨텍스트에서만 비교 데이터 반환.
- minGroupSize 같은 privacy floor 불필요(개별 선수 비교는 선수 화면에서 명시적으로 고른 것).

### 3.4 신기록(범위 명시) 보조 — 후순위
season-records가 이미 "모은 기록 빠른 순"이므로 1위 행이 곧 "모은 기록 중 1위".
별도 API보다 **카피/배지로** 처리(§4). 공식 신기록 연동은 별건.

## 4. UI/카피 설계 (Claude 도메인)

### 4.1 다종목 선수 — 종목 전환 (이번 1차 구현 대상)
- 프로필 상단에 종목 칩 그룹. `events.length >= 2`일 때만 노출.
- 칩 선택 시: 그 종목의 추이 그래프 + 베스트 + 변화량(사실값) 표시.
- 칩 라벨에 기록 수 뱃지 ("5000m · 8"). 가장 기록 많은 종목이 기본 선택.
- 카피: "종목을 골라 그 종목의 기록 흐름을 볼 수 있어요."

### 4.2 기간 토글
- "전체 / 최근 3년 / 최근 1년" 세그먼트.
- 기록 2개 미만이면 그래프 대신 "비교할 기록이 아직 적어요. 한 점만 있어요."

### 4.3 비교 화면 카피
- 제목: "기록 나란히 보기" (NOT "대결/순위").
- 고지: "AthleteTime이 모은 공개 기록을 나란히 둔 거예요. 순위·우열을 가리는 게 아니에요."
- 동명이인 고지 + 출처·대회·풍속 표기 유지.

### 4.4 신기록 표현 가드
- "AthleteTime이 모은 기록 중 최고" / "이 시즌 모은 기록 중 1위(모은 범위 안에서)".
- "한국신기록/공식기록/검증된" 금지.
