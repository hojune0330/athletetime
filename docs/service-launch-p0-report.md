# AthleteTime Service Launch P0 Report

## 목적

현재 호스팅 중인 AthleteTime에 바로 올릴 수 있는 기능만 남기고, 준비가 덜 된 기능은 "추후 오픈"으로 잠근다. 이번 패치는 서비스 첫인상과 운영 리스크를 동시에 줄이는 것을 목표로 한다.

## 이번에 열어두는 기능

| 영역 | 상태 | 이유 |
| --- | --- | --- |
| 홈 | 공개 | 서비스 목적과 현재 가능한 기능을 짧게 안내 |
| 대회/경기 결과 | 공개 | 기존 서비스 핵심 기능이며 공개 데이터 탐색의 1차 진입점 |
| 페이스 계산기 | 공개 | 개인정보·커뮤니티 리스크가 낮고 러너에게 즉시 유용 |
| 훈련 계산기 | 공개 | 개인정보·커뮤니티 리스크가 낮고 실사용 가치가 명확 |
| 로그인/회원가입 | 공개 | 인증 기반 기능 준비를 위해 유지 |

## 이번에 잠그는 기능

| 영역 | 상태 | 처리 |
| --- | --- | --- |
| 커뮤니티 | 추후 오픈 | 프론트는 Coming Soon, 백엔드 API는 503 `FEATURE_COMING_SOON` |
| 실시간 채팅 | 추후 오픈 | 프론트는 Coming Soon, 백엔드 HTTP/WS 모두 차단 |
| 중고거래 | 추후 오픈 | 프론트는 Coming Soon, 백엔드 API와 마이그레이션 차단 |
| 기록 검색/인사이트 | 추후 오픈 | 동명이인·출처·정정 흐름이 더 안정화된 뒤 공개 |
| 기록카드 | 추후 오픈 | 예시/가짜 화면 없이 실사용 흐름 기준으로 재정리 후 공개 |

## 보안·운영 변경

- 운영 환경에서 `JWT_SECRET`이 없으면 서버가 기동되지 않는다.
- 운영 환경에서 기본 JWT secret 값은 사용할 수 없다.
- 운영 CORS는 허용된 Origin만 통과한다.
- 이메일 인증 코드는 운영 로그에 노출하지 않는다.
- 커뮤니티/채팅/중고거래 API는 기능 플래그가 켜지기 전까지 503으로 응답한다.
- Netlify의 `/api/*` 프록시가 SPA fallback보다 먼저 적용되도록 순서를 고쳤다.

## 배포 전 필수 환경변수

### Backend Render

- `NODE_ENV=production`
- `JWT_SECRET`: 긴 무작위 문자열. 기본값 금지.
- `DATABASE_URL`: 운영 DB 연결 문자열.
- `FRONTEND_URL`: 실제 프론트 도메인.
- `RESEND_API_KEY`: 회원가입·비밀번호 재설정 인증 이메일 발송용. 운영에서는 필수.
- `CORS_ALLOWLIST` 또는 `ALLOWED_ORIGINS`: 추가 허용 도메인이 있으면 쉼표로 입력.

### Frontend Netlify

- `VITE_API_BASE_URL`은 비워두거나 설정하지 않는 것을 기본으로 한다.
- Netlify redirect가 `/api/*`를 Render로 프록시한다.
- 로컬 개발·preview에서는 `VITE_DEV_API_PROXY_TARGET`으로 `/api` 프록시 대상을 바꿀 수 있다.

## 검증 기준

- 프론트 `npm run build` 통과.
- 프론트 `npm run type-check` 통과.
- 프론트 `npm test`는 런칭 표면 자동 검증을 수행하고 통과.
- 백엔드 `npm run prebuild` 통과.
- 운영 JWT secret 누락 시 실패.
- 운영 JWT secret 설정 시 모듈 로드 성공.
- 운영 `RESEND_API_KEY` 누락 시 배포 검증 실패.
- 이메일 서비스 미설정 상태에서도 운영 로그에 인증 코드가 남지 않음.
- 미오픈 API는 503으로 닫힘.
- 미허용 CORS Origin은 403으로 거절.
- 프론트 주요 공개/잠금 라우트는 모두 200 SPA 응답.
- `/records`, `/profile-card`, `/community`, `/community/post/1`, `/write`, `/edit/1`, `/chat`, `/marketplace`, `/marketplace/new`는 브라우저 렌더 기준 Coming Soon으로 닫힘.
- 미오픈 프론트 라우트는 렌더 중 API/WS 요청을 만들지 않음.
- 백엔드 커뮤니티/채팅/중고거래 read/write 대표 API와 채팅 WebSocket은 기능 플래그가 꺼진 상태에서 닫힘.

## 현재 남은 품질 리스크

- `npm run lint`는 레포 전체 기존 부채로 실패한다. 이번 런칭 패치의 새 표면은 `npm test`, `npm run type-check`, `npm run build`, 브라우저 렌더 QA로 별도 검증했다.
- Vitest 패키지는 현재 설치되어 있지 않다. 기본 `npm test`는 런칭 표면 검증으로 전환했고, 기존 Vitest 진입점은 `npm run test:vitest`로 남겨 두었다.
- 페이스/훈련 계산기 내부 UX와 오래된 관리자 폼 코드는 이번 P0 범위에서 재설계하지 않았다.
