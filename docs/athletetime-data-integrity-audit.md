## ① 혼성경기 트랙 구간 + 계주의 eventKey 오염 & direction 오판 🔴

### 증상
- 김민상 상세: PB(indexedBest)=`5:17.37`, 시즌기록=`4:53.91`. **느린 기록이 "최고기록"으로** 표시.
- 2022 1500m 남자고등학교부 시즌표:
  - rank 1 최희태 `5:26.82`
  - rank 2 성지윤 `5:05.53`
  - rank 3 김승찬 `5:00.98`
  - rank 4 오창기 `3:58.29` ← **진짜 1등급 기록이 4위로 밀림**
  - rank 5 오민석 `3:58.35`
  - → **느린 10종 1500m 구간기록이 1위**, 정상 1500m가 하위로.

### 근본 원인
`normalizeEvent`가 `1500m(10종)` 라벨을 일반 `1500m`과 **같은 eventKey(`1500m`)** 로 합치는데,
혼성경기는 "점수 높을수록 좋음" 규칙(`higher`)을 그대로 트랙 구간(분:초)에 적용한다.

```
1500m       -> key: 1500m, direction: lower   (정상)
1500m(10종) -> key: 1500m, direction: higher  (오판 — 같은 키, 반대 방향)
```

### 오염 규모 (전수 측정)
**direction이 혼합(mixed)된 eventKey 7개** — 전부 우리 주력 종목:

| eventKey | 행수 | 섞인 라벨 |
|---|---|---|
| 100m | 6,032 | 100m + `4x100mR`(계주) + `100m(10종)` |
| 400m | 4,596 | 400m + `4x400mR` + `400m(10종)` |
| 1500m | 4,242 | 1500m + `1500m(10종)` + `1D/1E/2D 1500m` |
| 800m | 3,591 | 800m + `800m(7종)` + `1B/1C/1D 800m` |
| 200m | 2,975 | 200m + `200m(7종)` + `1C/1D/1E 200m` |
| 100m-hurdles | 1,222 | 100m 허들 (혼합 방향) |
| 110m-hurdles | 1,207 | 110m 허들 (혼합 방향) |

direction 명백 오판 행: **1,866행** (17개 라벨).
추가로 **계주(4x100mR, 4x400mR)가 개인 종목과 같은 키로 섞임** — 개인 100m 순위표에 계주 기록이 끼어든다.

### 권고 수정 (코덱스 영역)
1. 혼성경기 트랙 구간은 **별도 eventKey**로 분리: `1500m` ≠ `1500m-decathlon`(또는 `1500m(10종)` 그대로 유지).
2. **방향은 라벨이 아니라 종목 물리 단위로 결정**: 시간(트랙/허들/경보/마라톤/계주) = 무조건 `lower`,
   거리·높이(필드) = `higher`, 혼성경기 "총점"만 `higher`.
3. **계주를 개인 종목과 분리**: `4x100mR` → `4x100m-relay` 별도 키.

## ② 소속 표기 비정규화 → athleteKey 분열 + suppression 누수 🔴

### 증상 A — 동일인 카드 분열
같은 선수·같은 학교인데 시·도 접두 표기가 달라 **여러 athleteKey로 쪼개짐**:
```
선수A :: 전남체육고등학교 | 전남-전남체육고등학교 | 전라남도-전남체육고등학교   (3개 카드)
선수B :: 한국전력공사 | 전북-한국전력공사 | 전라남도-한국전력공사            (3개 카드)
선수C :: 충북-괴산군청 | 충청북도-괴산군청                                  (2개 카드)
```
- athleteKey = `stableId(name|team)` 이므로 team 표기가 다르면 다른 키.
- 결과: 동일인의 PB·기록 발자취가 카드별로 분산되어 "내 기록 한눈에"가 불완전.

### 증상 B — suppression(비공개) 누수 (신뢰/프라이버시 직결)
`대구광역시-대구체육고등학교` 선수D를 `hide` 처리해도, 데이터에 존재하는
`대구체육고등학교` / `대구-대구체육고등학교` 표기의 동일인 기록은 **검색에 그대로 노출**.
- `checkSuppression`은 `athleteName + affiliation` 정확 일치 → 표기 변형 1개만 막고 나머지 누수.
- 우리가 공들인 `search_hidden` de-index 정책의 실제 보호 범위에 구멍.

### 규모 (정밀 측정)
- "같은 이름 + 시·도 접두만 다른 동일 핵심 소속" 분열: **5,861 그룹, 14,058 카드** (전체 24,986 중 ~56%).
- (주의) 이 수치는 진학/이적으로 소속이 바뀐 정상 케이스는 제외한, 표기 변형만 카운트한 값.

### 권고 수정 (코덱스 데이터 + 클로드 신뢰)
1. **소속 정규화 함수** 도입: 시·도 접두(`전남-`, `전라남도-`, `대구광역시-` …)를 표준형으로 통일.
2. athleteKey를 **정규화된 team 기준**으로 재산정 (단, 진학/이적은 합치지 않도록 학교 본체는 보존).
3. `checkSuppression` / suppression 매칭을 **정규화 키 기준**으로 → 표기 변형 전체를 한 번에 차단.
4. (클로드) 동명이인/표기 변형 안내 카피: "같은 이름의 다른 선수일 수 있어요. 소속·연도를 확인해 주세요."

## ③ `공던지기` direction 오판 🟠

```js
const FIELD_HINTS = /(높이|멀리|세단|장대|포환|원반|창던|해머|HJ|LJ|TJ|PV|SP|DT|JT|HT)/i;
```
- `창던`(창던지기)은 있지만 **`공던`(공던지기)이 누락** → 던지기인데 `lower`로 오판.
- 공던지기는 초등부 종목으로 멀리 던질수록 좋음(`higher`)이어야 함. 61+44건.
- 수정: FIELD_HINTS에 `공던` 추가 (또는 `던지기` 일반 패턴 + 예외 처리).

## ④ `역전경기`(에키덴/역전마라톤) 파싱 깨짐 🟡

비교가능(comparable)인데 recordValue<=0인 33건 조사 결과, `역전경기 1일차` 데이터에서
**선수 이름과 기록 필드가 뒤섞여** 파싱됨:
```
recordDisplay="고지우0  0"  name="김동혁0  0"  eventLabel="역전경기 1일차"
recordDisplay="이동호0  0"  name="송성재25:57  1:21:47"
```
- 역전경기는 구간 릴레이 특수 포맷이라 일반 행 파서로는 안 됨.
- 수정: `역전경기`/`역전마라톤`을 비교가능 분석에서 제외(또는 전용 파서). 33건이라 영향은 작지만 무결성 이슈.

## 검증 메모 (재현 방법)
- analytics 4개 엔드포인트 모두 200 OK (filters/records.search/athletes.:key/season-records).
- suppression 동적 검증: 정확 키 hide 시 summary는 차단 ✅, but 표기 변형 카드는 누수 ⚠️ (②).
- 프론트 빌드: `✓ built in 14.63s` (chunk-size 경고만, 비차단).
- 모든 검증은 데이터/HTTP/동적 호출로만 수행, 운영 파일(suppressions.json) 즉시 원복.
