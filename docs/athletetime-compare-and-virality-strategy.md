## 0. 핵심 긴장 (먼저 명시)

선생님 지시 3번에는 우리 프로젝트의 **근본 원칙과 정면으로 부딪히는** 요소가 있다.

| 선생님이 원하는 것 | 우리의 불변 원칙 | 충돌 |
|---|---|---|
| 자랑할 만한 콘텐츠 | "공식/랭킹/평가/우열" 단정 금지 | 자랑 = 보통 "남보다 우월" 프레임 |
| 공격적 확산, 많은 사람이 보게 | "모은 범위 안에서"만 말하는 절제 | 자극적일수록 단정적이게 됨 |
| 조금 네거티브해도 OK | 선수 인격·기록은 민감정보 | 망신·비교당함·낙인 리스크 |

**이 문서의 입장:**
확산은 추구하되, **"남을 깎아내리는 경쟁"이 아니라 "자기 성취를 자랑하는 자부심"** 으로
바이럴 엔진의 방향을 튼다. 즉 **네거티브 에너지를 사람(타인 비교/순위)이 아니라
"기록의 드라마"(내 성장폭·연속 출전·다종목 도전 등 자기 서사)** 로 흡수시킨다.
이렇게 하면 "공격적 확산"과 "신뢰/비랭킹"을 동시에 만족할 수 있다.

> 한 줄 원칙: **"누가 이겼나"가 아니라 "내가 얼마나 왔나"를 자랑하게 한다.**

## 1. 파트 A — 선수 비교(Compare) UX 설계

### 1.1 두 가지 진입 방식

선생님이 말한 "직접 비교"와 "여러 명 선택 비교"를 두 모드로 분리한다.

**(1) 직접 비교 (1:1, 빠른 진입)**
- 선수 프로필(AthletePanel) 안에 "이 선수와 비교" 버튼.
- 누르면 같은 종목을 가진 다른 선수를 검색 → 1명 추가 → 나란히.
- shadow-cluster의 "같은 선수 추정"과 헷갈리지 않게 **다른 선수 비교는 명시적으로만**.

**(2) 다중 선택 비교 (담기 → 비교판)**
- 검색 결과/시즌표 각 행에 **"비교에 담기"(➕) 체크**.
- 화면 하단 고정 **"비교 트레이(tray)"** 에 담긴 선수 칩이 쌓임 (최대 4~6명).
- "비교하기" 누르면 **나란히 보기 화면**으로.
- 트레이는 localStorage 유지(세션 넘어 담아두기 가능).

```
검색결과 ─┐
시즌표  ─┤→ [➕ 비교에 담기] → ╔═ 비교 트레이 (하단 고정) ═╗ → [나란히 보기]
선수카드 ─┘                     ║ 김OO ✕  이OO ✕  +담기  ║
                               ╚════════════════════════╝
```

### 1.2 비교 화면 = "기록 나란히 보기" (NOT 대결)

핵심 제약: **같은 종목·같은 단위 안에서만** 비교. 단위 다른 종목 혼합 금지.

레이아웃:
- 상단: 종목 선택(공통으로 가진 종목만 후보), 기간(전체/3년/1년).
- 본문: 선수별 컬럼 — 시즌 베스트 / 모은 기록 중 최고 / 기록 수 / 출전 연도 폭.
- 중앙: **하나의 추이 그래프에 여러 선수 라인 오버레이**(같은 종목·단위이므로 가능).
  - 라인별 색 + 범례. 점선/실선이 아니라 색으로만 구분(우열 암시 금지).
- 하단: 출처·대회·풍속(legal) 표기 + 동명이인 고지 + 신뢰 고지.

카피(확정 톤):
- 제목: **"기록 나란히 보기"**
- 고지: "AthleteTime이 모은 공개 기록을 나란히 둔 거예요. **순위·우열을 가리는 게 아니에요.**
  같은 이름이라도 다른 선수일 수 있고, 풍속·대회 조건이 달라요."
- 금지: "승자/1위/이김/우세/추월/역전" 같은 대결 단어. → "더 빠른 기록을 모았어요" 정도의
  **사실 진술**만, 그것도 "모은 범위 안에서".

### 1.3 비교에서의 신뢰 가드 (반드시)
- 공통 종목이 없으면 비교 불가 안내("두 선수가 함께 가진 종목이 없어요").
- 비교 가능 기록이 한쪽이라도 2개 미만이면 그래프 대신 표만 + "흐름 보기엔 기록이 적어요".
- 풍속 불법(windLegal=false) 기록은 배지로 구분("참고용·풍속 초과").
- 미성년/학생 선수 다수 → **"비교는 응원·관전용이며 평가가 아니에요"** 상시 노출.

### 1.4 API (Codex 도메인 — 기존 제안 §3.3 재확인/보강)
```
GET /analytics/compare?eventKey=&divisionKey=&season=&athleteKeys=k1,k2,k3
→ { eventKey, eventLabel, season?, divisionLabel?,
    athletes: [{ athleteKey, name, team, ambiguity,
                 seasonBest, indexedBest, recordCount, fromYear, toYear,
                 trail: TrailPoint[] }],
    commonEvents: [{eventKey,eventLabel}],   // 공통 종목 후보 (NEW)
    disclaimer }
```
- 다중 athleteKeys 지원(2~6). 공통 종목 계산은 서버가.
- 읽기전용·person_no 미사용 유지.

## 2. 파트 B — 확산(바이럴) 전략

### 2.1 전략의 축: "자랑은 자기 성취로, 비교는 응원으로"

확산 콘텐츠를 **3개 톤**으로 설계하고, 네거티브 에너지를 안전한 쪽으로 유도한다.

| 콘텐츠 | 자랑 포인트 | 누가 공유? | 네거티브 흡수 방향 |
|---|---|---|---|
| **A. 성장 카드** | "3년간 +1.4초 / -12초 성장" | 본인·코치·부모 | 타인이 아니라 **과거의 나** 와 경쟁 |
| **B. 다종목 도전 카드** | "한 해 7종목 출전" | 본인·팀 | "다재다능"은 우열 무관, 순수 자부심 |
| **C. 연속·꾸준함 카드** | "5년 연속 출전 / 14개 대회" | 본인·관계자 | 성실 서사 — 깎아내릴 대상 없음 |
| (보조) 나란히 보기 공유 | "라이벌과 같은 트랙" | 팬·동료 | "대결" 아닌 "함께 달린 기록" |

> 핵심: **자랑의 기준점을 "남"이 아니라 "과거의 자신/자신의 폭"으로 옮긴다.**
> 그러면 자극적·공유욕구는 살리되, 타인 망신/랭킹 단정 리스크는 차단된다.

### 2.2 "자랑할 만한" 공유 카드 (Shareable Brag Card)

이미 프로젝트에 카드 스튜디오(기록카드) 인프라가 있음(html2canvas/jspdf 번들 존재).
이를 **공유 최적화 카드**로 확장.

- **포맷:** 정사각(인스타) + 세로(스토리/릴스) 두 비율. 다운로드 + "이미지 복사".
- **요소:** 선수명·소속(자기 선택)·종목·성장폭(사실값)·기간·작은 추이 미니그래프·출처 워터마크 "AthleteTime이 모은 공개 기록".
- **자동 카피 생성(신뢰 가드 통과한 것만):**
  - ✅ "2021→2024, 5000m에서 모은 기록이 18초 줄었어요" (사실·범위 명시)
  - ✅ "올해 7종목 출전 기록을 모았어요"
  - ❌ "전국 1위" / "한국신기록" / "최강" (금지)
- **워터마크/링크:** 카드마다 해당 선수 검색 딥링크(`/records?q=`) 자동 삽입 → 보는 사람이 바로 유입.

### 2.3 확산 루프 (Growth Loop) 설계

```
검색·발견 → 자기 기록 확인 → 성장/다종목/꾸준함 발견(놀람)
   → 자랑 카드 생성(자부심) → SNS 공유(워터마크+딥링크)
   → 지인/팬 유입(검색) → 그들도 자기/응원 선수 검색 → 반복
```
- 트리거(놀람 포인트)를 **자동으로 표면화**: 프로필 진입 시
  "올해 가장 많이 줄어든 종목" / "가장 오래 출전한 종목" 같은 **하이라이트 배지**를
  먼저 보여줘서 "어 이거 자랑하고 싶다"를 유발.
- 공유 카드에 딥링크가 박혀 있어 **공유 = 신규 유입**.

### 2.4 관계자·팬을 끌어들이는 장치
- **응원 비교(나란히 보기) 공유:** 팬이 "내 최애 선수 둘 같은 트랙" 카드를 만들어 공유.
  대결이 아니라 "둘 다 응원" 프레임 카피 강제.
- **팀/학교 단위 하이라이트:** "OO고 올해 모은 기록 하이라이트"(division 축 활용) — 코치·학교가 공유.
- **대회 직후 타이밍:** 시즌·대회 키가 있으니 "이번 대회에서 모은 내 기록 카드" 즉시 생성 유도.

### 2.5 "조금 네거티브해도 OK"를 안전하게 해석
선생님 단서를 **남을 깎는 네거티브가 아니라, "분발/자극"의 네거티브**로 한정한다.
- ✅ 허용: "작년보다 0.3초 느려진 기록도 있어요"(본인 한정 사실) — 본인이 분발 동기로 공유.
- ✅ 허용: "이 종목은 최근 기록이 적어요"(자기 데이터 공백 자각).
- ❌ 금지: 타인 대비 "하위권/뒤처짐/추락" 낙인, 자동 순위표 망신, 동의 없는 비교 푸시.
- 안전장치: 부정적 사실값도 **본인 카드에서는 본인이 선택**해 노출/공유(opt-in). 타인 카드엔 미노출.

## 6. 결정 확정 (선생님 회신 2026-06-12 반영)

> 아래 4개는 **확정**. 이 절을 기준으로 P1 구현.

### 6.1 확산 톤 — **승인 ✅**
- "자랑 = 자기 성취/성장폭(타인 비교 아님)" 프레임으로 확정.
- "공격적 확산"은 살리되 "남 깎기/타인 순위 단정"은 배제. (이 문서 §0·§2.1 그대로)

### 6.2 부정 사실값 — **모두 노출 + 공유 시 본인 선택(opt-in)으로 진화 ✅**
- **기본 화면(프로필/추이/비교)에서는 모든 사실값을 그대로 노출.** (느려진 기록도 숨기지 않음 — 데이터 정직성)
- 단, **"공유용 모아보기/내보내기" 시점**에는 **노출할 기록·기간·종목 범위를 본인이 선택**할 수 있게 한다.
  - **자기 기록에 한해서만** 선택 가능. (타인 카드/타인 기록은 범위 선택 불가 → 동의 없는 편집·왜곡 방지)
  - 즉 "보기"는 정직(전체), "공유"는 본인 큐레이션(선택)으로 분리.
  - 구현 형태: 공유 카드 만들기 단계에서 "포함할 기록 고르기"(체크) + "기간/종목 범위" 선택 UI.
- 안전장치: 선택해도 **사실 왜곡 카피는 금지**(빼는 건 가능, 거짓 추가는 불가). 워터마크·범위 고지는 제거 불가.

### 6.3 비교 인원 — **1:다수, 보수적 상한 ✅**
- 1:1이 아니라 **1명 기준 + 비교 대상 추가** 형태(1:N).
- **상한: 최대 4명**(기준 1 + 추가 3). 보수적 느낌 유지.
  - 근거: 라인 오버레이 그래프 가독성(색 구분 한계), 모바일 컬럼 폭, "대결판"으로 비대해지지 않게.
  - 트레이 담기도 동일 상한(4)로 캡. 초과 시 "비교는 최대 4명까지 나란히 볼 수 있어요" 안내.

### 6.4 워터마크 + 출처 — **워터마크 필수, 출처는 "대회+날짜" 표기 ✅**
- **워터마크(필수, 제거 불가):** `AthleteTime · 모은 공개 기록 · 공식 기록 아님` + 검색 딥링크(`/records?q=`).
- **출처(함께 표기):** 데이터 실측상 `source.sourceUrl`/`capturedAt`는 **전부 비어 있고**,
  provider는 일괄 **`KAAF`(대한육상연맹)**. 따라서 출처는 URL이 아니라
  **"제공: 대한육상연맹 공개 결과 · {대회명} ({날짜})"** 형태로,
  카드에 실제 기록이 나온 **대회명·날짜**를 함께 새긴다. (신뢰·검증가능성↑, 동시에 "공식 인증 아님" 유지)
  - 여러 기록을 모은 카드면 대표 대회 1~2개 + "외 N건" 표기.
  - 풍속 초과(windLegal=false) 포함 시 "일부 참고용(풍속 초과)" 배지 동반.

> **확정 완료.** 아래 §7 구현 계획(P1)대로 진행한다.
