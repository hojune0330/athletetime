§2 · Three pillars

왜 다른가

세 가지 기둥이 TRAINORACLE을 지탱합니다.

P−1 · Why-first

모든 답에 근거가 따른다

"오늘 6×1000m @ 3'20"" 라는 권장 뒤에는 5000m PB 16:10 기준 VO2 페이스 환산이라는 근거가 항상 따라옵니다. 출처는 클릭 가능한 인용 형태로 명시됩니다.

P−2 · Honest uncertainty

모르면 모른다고 말한다

두 신호가 충돌하면 "판정 불확실"로 표기하고 코치 판단을 요청합니다. 신뢰도 70% 미만은 자동으로 인박스로 이관. AI는 결코 100% 확신을 가장하지 않습니다.

P−3 · 9.5-day cycle

달력이 아닌 회복 주기

주간 단위는 달력의 산물입니다. 30년의 코칭 데이터에서 추출한 회복–자극 평균 주기는 9.5일. 이 주기를 따른 선수들의 시즌 PB 갱신율은 +18% 높았습니다.

§3 · Features

일하는 방식

개념이 아닌 실제 화면. 매일 사용하는 도구로서 어떻게 작동하는지.

F · 01 — Calendar

9.5일 사이클이 캘린더의 기본 단위

Week / 9.5-Cycle / Timeline 3가지 뷰. 9.5-Cycle 뷰는 다른 어디에도 없는 우리만의 화면입니다. MAIN 세션을 중심으로 전후 회복·보조 세션이 자동 배치됩니다.

코치는 Timeline 뷰로 12주 전체를 한눈에 보면서 대회 D-day까지의 페이즈를 확인할 수 있습니다.

F · 02 — 장호준 AI

매 답변에 출처와 신뢰도

장호준 AI는 단순 chatbot이 아닙니다. 모든 응답에 verdict(확정 / 권장 / 판정 불확실 / 데이터 부족)와 신뢰도 %가 표시됩니다.

주장 뒤에는 인용 번호 [1]가 붙고, 클릭하면 우측 패널에서 출처를 확인할 수 있습니다. 코치 노트, 외부 문헌, 본인 과거 세션, 모두 검증 가능한 근거.

F · 03 — AI Inbox

코치의 우선순위 큐

AI가 발견한 모든 신호 — 판정 불확실, 부상 위험, 패턴 변화, 규칙 위반, 규칙 통과 — 가 우선순위별로 정렬됩니다.

코치는 8명의 선수를 동시에 보면서, AI가 미리 추려낸 3 unread 항목만 처리하면 됩니다. 100명을 동시에 보는 것이 가능해지는 이유.
