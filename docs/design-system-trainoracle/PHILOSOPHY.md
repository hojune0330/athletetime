## 1. 제품 정체성

### 한 줄 정의
> **"Strava가 기록하고, TrainingPeaks가 분석한다면, TRAINORACLE은 생각합니다."**

### 무엇이 아닌가
- ❌ 일반 피트니스 앱이 아닙니다 (마스 마켓 X)
- ❌ 소셜 운동 플랫폼이 아닙니다 (좋아요·팔로우 X)
- ❌ 의료 진단 도구가 아닙니다 (disclaimer 필수)
- ❌ 게이미피케이션 도구가 아닙니다 (배지·레벨 X)

### 무엇인가
- ✅ 엘리트 육상 코치/선수를 위한 **thinking tool**
- ✅ 코치 장호준의 30년 방법론을 **AI로 확장**한 도구
- ✅ 매 결정에 **근거와 신뢰도**를 명시하는 정직한 도구
- ✅ 1500m–10000m **중장거리 종목** 특화

## 3. 디자인 원칙 (10 Rules)

### 3.1 Mobile-first (360px)
모든 새 컴포넌트는 360px 너비에서 먼저 작동해야 합니다. Desktop은 점진적 확장.

### 3.2 Scientific, not sporty
신경의학·생리학 연구 도구의 무게감. **운동복 광고 같은 색감 금지**. 채도가 높은 색은 절대 사용하지 않습니다.

### 3.3 Data-dense yet breathable
화면당 정보량은 많아야 하지만 (코치는 빠른 의사결정 필요), 여백과 계층으로 숨 쉬게 합니다. **카드로 묶어서 압축하는 게 아니라, 질서로 정렬**합니다.

### 3.4 Evidence-first
모든 추천 뒤에는 근거가 따라옵니다. "오늘 6×1000m"가 아니라 "오늘 6×1000m **(왜? — VO2 페이스 환산 + Cycle 7의 D5)**".

### 3.5 Honest uncertainty
- AI는 100% 확신을 가장하지 않습니다
- 신뢰도 < 70% → 자동으로 코치 인박스 이관
- 두 신호 충돌 → "판정 불확실" verdict
- 데이터 부족 → "데이터 부족" verdict
- 답변에 **반대 의견**도 함께 제시 (한쪽 의견만 강요 X)

### 3.6 One-thumb operable
모바일은 한 손 조작 가능해야 합니다. 핵심 액션은 화면 하단 sticky bar에. 터치 타깃 **44×44px 이상**.

### 3.7 Pro-tools speed
키보드 단축키, command palette (⌘K), 빠른 검색. **클릭 수 최소화**.

### 3.8 Preserve terminology
훈련 용어는 **영어 원문 유지**. CK, EPOC, TSS, TSB, RPE, HR drift, VO2max, LT, Z1–Z6, MAIN, AUX, BASE, LT, VO2, GLY, ATP, A_guide, 9.5-day cycle 등.

UI 라벨/설명은 **한국어**, 측정값/페이스/시간은 **모노스페이스 영문**.

### 3.9 Three calendar views
Week / 9.5-Cycle / Timeline. **9.5-Cycle 뷰가 핵심 차별화**. Week만 만들면 일반 트레이닝 앱과 차이 없음.

### 3.10 Silence over noise
알림 폭탄 금지. AI는 정말 중요한 것만 인박스로 보냅니다. 무의미한 축하 메시지, 동기부여 푸시 알림 금지.

## 4. 9.5일 사이클 — 제품의 심장

### 왜 9.5일인가
7일은 달력의 산물. 132명 엘리트 선수의 9년 데이터에서 추출한 회복–자극 평균 주기는 **9.5일**.

### 사이클 구조
```
D-1: 회복 (recovery)
D-2: Z2 base
D-3: Z2 base
D-4: Z2+ base+
D-5: ★ MAIN (VO2 또는 LT) ★ 사이클의 정점
D-6: Z1 회복
D-7: LT 보조
D-8: Z2 base
D-9: 휴식 (full rest)
D-.5: 다음 사이클 D-1과 overlap (transition)
```

### 9 Rules — 자동 검증되는 규칙

| Rule | 이름 | 검증 방법 |
|---|---|---|
| R-1 | 회복 우선 원칙 | D-1, D-6에 강도 자극 없음 |
| R-2 | D-5 MAIN 고정 | MAIN은 D-5 ±1일만 허용 |
| R-3 | Z1 회복 비율 ≥20% | 사이클 총 시간의 20% 이상 |
| R-4 | 주관 신호 우선 | RPE/Sleep > CK/HR drift |
| R-5 | Volume × Intensity 트레이드 | 한 사이클에 둘 중 하나만 ↑ |
| R-6 | VO2 MAIN 볼륨 5–8 km | 미달/초과 시 경고 |
| R-7 | 드리프트 ±3" 관찰 | MAIN 마지막 2 rep |
| R-8 | 2주 평균 우선 | 단일 데이터로 판정 X |
| R-9 | 대회 D−21 테이퍼 | 자동 볼륨 감소 |

각 규칙은 매 사이클마다 자동 검증 → Validation 화면에 표시됩니다. 위반 시 AI 인박스로 자동 이관.

## 5. AI의 역할 — "thinking tool"

### AI는 코치를 대체하지 않는다
- AI는 100명의 데이터를 동시에 본다
- 코치는 1명의 영혼을 본다
- **둘 다 필요하다**

### Verdict 시스템 (4단계)
| Verdict | 의미 | 사용 시점 |
|---|---|---|
| **CONFIRM** | 확정 (사실 기반) | 측정값 환산, 단순 계산 |
| **RECOMMEND** | 권장 (근거 있음) | 페이스, 강도 추천 |
| **UNC** | 판정 불확실 | 신호 충돌, 신뢰도 < 80% |
| **LACK** | 데이터 부족 | 14일 미만 데이터 |

**모든 AI 응답은 위 4가지 중 하나의 verdict와 신뢰도(%)를 가져야 합니다.**

### Citation 의무
- AI의 모든 주장에 출처 인용 `[1] [2]`
- 출처 종류: Coach Jang 노트, 외부 문헌(Daniels 등), 본인 과거 데이터, 엔진 노트
- 클릭 시 사이드 패널에 출처 펼침

### "다른 관점" 섹션
모든 응답에 **반대 의견** 또는 **대안적 해석**이 함께 제시되어야 합니다. AI가 일방적으로 한 답만 강요하면 코치의 판단력을 약화시킵니다.

## 6. 콘텐츠 톤

### 텍스트 작성 원칙
- **공식적 한국어** (반말 X, 너무 부드러운 ~요체도 자제)
- **숫자는 정밀하게** (3'20"/km, HR 178–186)
- **추측 표현 자제** ("아마도", "~ 일 수 있다" → "데이터에 따르면", "권고 기준")
- **격려·축하 메시지 금지** ("잘했어요!", "화이팅!" 등 사용자 인풋 없으면 금지)
- 마이크로카피는 **모노스페이스로 메타 정보** 표시 (시간, 단위 등)

### 한국어/영어 혼용 규칙
| 카테고리 | 언어 | 예 |
|---|---|---|
| UI 라벨 | 한국어 | 대시보드, 캘린더, 분석 |
| 일반 메시지 | 한국어 | "좋은 아침이에요" |
| 훈련 용어 | 영어 | VO2, LT, BASE, MAIN, AUX |
| 측정 지표 | 영어 + 모노 | HR 178, TSS 124 |
| 페이스/시간 | 영어 + 모노 | 3'20"/km, 16:10.44 |
| 규칙 코드 | 영어 + 모노 | R-6, D-5, T-2 |

## 7. 엔지니어링 원칙

### 정확성 > 속도
숫자가 잘못 표시되면 안 됩니다. 페이스, HR, TSS 모두 검증된 공식으로 계산.

### Honest API
- AI 응답에 confidence < 0.5 → throw error 또는 LACK verdict 반환
- 데이터 부족 시 → throw informative error, fallback 추정값 사용 시 **명시**

### 시간/날짜
- 모두 **UTC + KST 표시** (선수가 해외 대회 갈 때 대비)
- 사이클의 D-day는 **선수 로컬 타임존** 기준

### Privacy by default
- 다른 코치/선수의 데이터 노출 금지
- AI 학습에 개인 데이터 사용 금지 (RAG는 추론만)
- 데이터 내보내기·삭제 권한 사용자에게 항상 제공

### ✅ 좋은 예 1
> Today's session: **Recovery (Z1)** · 30 min
> Why: Cycle 7, Day 6/9.5 — D-5 MAIN session(VO2) follow-up. CK +18% over 2wks. [Rule 4]

### ❌ 나쁜 예 2
"AI가 분석한 결과 오늘은 무리하지 않는 게 좋겠어요." — 막연함, 근거 없음, 신뢰도 없음

### ✅ 좋은 예 2
> **Verdict: 판정 불확실** (신뢰도 72%)
> CK +18% (above ±8% band) — fatigue marker raised.
> However, RPE stable (5.8/10) and HR drift 2.1% (within band).
> A_guide Rule 4 [1] suggests subjective signals override objective markers in conflict.
> **Recommend: −10% intensity** (3'20" → 3'25").
> Alternative view: Daniels [2] would suggest −15% based on CK alone.
> ▶ Coach decision required.
