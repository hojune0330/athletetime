# 작업 지시서 — 트랙 J: /records 단계형(토스식) 재편 — "내 기록 찾기"와 "기록 둘러보기" 분리

- 발행: 2026-07-09, Claude(총괄) — UX/UI 사양 확정본(임의 변경 금지, 이견은 PR 코멘트로 질문)
- 수행: Codex
- 배경(사장님 지시 요지):
  1. **내 기록 찾기**와 **일반 기록 검색**을 분리할 것.
  2. 내 기록 선택이 "하나 고르면 위로 붙고, 다음 걸 고르려면 다시 아래로 스크롤" — 번거로움. **토스처럼 한 단계=한 화면**으로 진행할 것. 스크롤 강제 금지.
  3. 일반 검색(다른 사람/다른 팀/시즌 현황)도 **단계를 줘서** 한 페이지에 흩뿌리지 말 것.
  4. 다음 단계까지 고려해 둘 것.

## 1. 정보 구조 (확정)

`/records` 는 **허브**가 되고, 두 개의 흐름으로 갈라진다.

```
/records                          ← 허브: 2택 + (담긴 게 있으면) 내 기록 요약 바
├─ /records/mine   (flow=mine)    ← 흐름 A: 내 기록 찾기 (위저드)
└─ /records/browse (flow=browse)  ← 흐름 B: 기록 둘러보기 (선수/팀/시즌표)
```

- 라우팅은 실제 서브라우트(`/records/mine`, `/records/browse`) 또는 `?flow=&step=` 쿼리 중 **구현이 단순한 쪽**을 택하되, 다음 3가지는 절대 조건:
  - (a) 브라우저 뒤로가기 = 이전 단계로 이동 (히스토리에 단계가 쌓여야 함)
  - (b) 기존 공유 링크 `?athlete=<key>` / `?compare=a,b` 는 **그대로 동작** (허브를 거치지 않고 해당 화면 직행)
  - (c) 새로고침 시 현재 단계 유지

## 2. 허브 화면 (확정 사양)

- 상단: 기존 히어로 카피 유지("내 기록, 이름만 알면 찾아요.")
- 본문: **큰 선택 카드 2장** (세로 스택, 모바일 우선):
  1. **내 기록 찾기** (primary, brand 배경 강조) — 부제: "이름만 입력하면 흩어진 내 기록을 모아드려요."
  2. **기록 둘러보기** (outline) — 부제: "다른 선수·팀·시즌 순위를 단계별로 찾아봐요."
- 내 기록이 이미 담겨 있으면(myEntries>0) 카드 1 위에 기존 슬림 요약 바 유지 — 탭하면 흐름 A의 완료 화면(A-4)으로 직행.
- 허브에는 검색창을 두지 않는다(흩뿌림 금지). AnonymousInsightCards는 카드 2장 아래로 이동.

## 3. 흐름 A — 내 기록 찾기 (토스식 위저드, 확정 사양)

**원칙: 한 화면 = 한 질문 = 한 결정. 화면 안에서 아래로 스크롤해야 다음 행동이 나오는 구조 금지. 주 CTA는 화면 하단 고정(sticky).**

- 공통 프레임: 상단 progress dot(1→4) + "뒤로" + 우측 "그만두기"(허브로).

### A-1. 이름 입력
- 큰 단일 입력창(autofocus), 안내 "선수 이름을 입력하세요 (두 글자 이상)", 보조 문구 "소속으로도 찾을 수 있어요".
- CTA: "내 기록 찾기". 입력 2자 미만이면 비활성.

### A-2. 후보에서 나 고르기 (핵심 수술 지점)
- 검색 결과를 **선택형 리스트**로 표시: 각 행 = 이름·소속·연도 범위·종목 요약 + **체크 토글**.
- **다중 선택을 이 화면 하나에서 끝낸다.** 지금처럼 "하나 누르면 위로 이동 → 다시 스크롤" 패턴 폐지. 선택해도 리스트는 제자리, 행에 ✓만 표시.
- 하단 고정 바: "n개 선택됨 · 다음". 0개면 비활성.
- 헷갈림 방지 카피: "같은 이름이 여러 명일 수 있어요. 소속·연도를 보고 내 것만 고르세요."

### A-3. 같은 사람 묶음 확인
- 선택한 후보들에 대해 기존 EstimatedSameAthleteCard 로직(동일 인물 추정 묶음)을 **한 화면 요약**으로 제시:
  - "이 기록들도 회원님 것 같아요" + 추정 묶음 리스트(기본 전체 체크 ON, 개별 해제 가능).
  - 추정 묶음이 없으면 이 단계는 **자동 스킵**.
- CTA: "이대로 합치기".

### A-4. 완료(내 기록 홈)
- 요약 카드: 이름(대표), 담긴 묶음 n개, 최고기록(PB)·최근 기록·시즌 순위 한 줄("2026 남자 고등부 100m 12위" 형식 — divisionLabel 재사용).
- 행동 3개(버튼, 스크롤 없이 보이는 위치):
  1. "기록 더 추가하기" → A-1로 (기존 선택 유지 상태로 재진입)
  2. "시즌표에서 내 위치 보기" → 흐름 B 시즌표로 내 행 하이라이트 진입(highlighted row 재사용)
  3. "기록 카드 공유" (기존 공유 카드 재사용)
- 묶음 개별 빼기(remove)는 이 화면의 목록에서만.

## 4. 흐름 B — 기록 둘러보기 (확정 사양)

### B-0. 무엇을 볼까요? (3택 카드)
1. **선수 찾기** — "다른 선수의 공개 기록"
2. **팀(소속)으로 찾기** — "소속별 선수 목록"
3. **시즌 기록표** — "종목·부문별 순위"

### B-1a. 선수: 검색 → 결과 리스트 → 프로필(기존 AthletePanel 재사용)
- 여기서의 행동은 **보기/비교 담기**가 기본. "내 기록이에요" 버튼은 프로필 화면에 보조 버튼으로만 남긴다(누르면 흐름 A-4로 전환·합산). 검색 결과 리스트에서는 제거.

### B-1b. 팀: 소속 검색 → 해당 소속 선수 리스트 → 프로필
- 백엔드는 기존 `records/search`가 소속 매칭을 이미 지원하므로 재사용. 소속 그룹핑 UI만 추가(같은 team 값끼리 묶어 헤더 표시).

### B-1c. 시즌표: 단계 select — ①시즌 ②종목 ③성별 ④층위 → 표
- 트랙 I의 2단 필터·defaultSeasonSelection·고정 순서를 그대로 사용하되, 첫 진입만 단계형으로 받고 표 화면 상단에는 현재 선택 요약 칩(탭하면 해당 단계로 되돌아감)을 둔다.
- 비교 트레이·CompareView 동작 불변.
