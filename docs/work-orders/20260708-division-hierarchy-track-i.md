## 1. 현재 결함 (재현 확인됨)

시즌 기록표 부문 드롭다운이 75개 항목이고 그중:

- `남자부`가 **16개 중복** (`남자부`, `kaaf-kind-10`, `kaaf-kind-12`, `kaaf-kind-1c`, …)
- `여자부` 16개, `남자고등학교부` 4개, `여자고등학교부` 5개 중복
- 정렬이 가나다순이라 U18이 맨 위, 일반부가 한참 아래

### 원인

1. **키 이원화**: TOP100 행은 `normalizeCandidateTopEvent()`가
   `divisionKey = kaaf-kind-${kindCd}`를 쓰고, data/results 행은
   `stableSlug(라벨)`을 쓴다. 같은 "남자부"가 서로 다른 키 17개로 존재.
2. **kindCd는 부문 코드가 아니다**: 데이터 검증 결과 kindCd 10에
   남자부+일반부+대학부+고등부가 혼재, 13에 고등부+남자부 혼재 등 —
   부문 판별에 사용 불가. **kindCd 기반 divisionKey는 폐기한다.**
3. **정렬 정책 부재**: `label.localeCompare` 가나다순뿐.
4. **원천 라벨 파편화**: data/results에 56종 라벨
   (남자부/남자일반부/남자고등학교부/M45/남자중학교 2학년부/MA/통합부 …).

### 결과적 사용자 피해

시즌 최고기록 버킷이 키 단위로 쪼개져 **같은 부문의 시즌 1위가 버킷마다
다르게 보인다.** 이것이 이 트랙의 수정 대상이다.

## 2. 층위 모델 (사장님 확정 정책 — 그대로 구현)

핵심 원칙: **"층위는 살리되, 무분별하게 묶거나 순서 매기지 않는다."**

### 2.1 2계층 구조

```
성별 (최상위 롤업)          소속 층위 (하위)
─────────────────          ──────────────────────────────
남자부  ← 남자의 모든        일반부   = 실업팀 + 무소속 (통칭 '실업')
        하위 층위를 흡수     대학부
여자부  ← 여자의 모든        고등부
        하위 층위를 흡수     중학부
혼성    (혼성계주 등)        초등부
                            U20 / U18 (연령 기반 — 별도 유지)
                            마스터즈 (M/W 연령대)
                            층위 미상 (원천에 성별만 있는 경우)
```

- **남자부/여자부 = 성별 롤업**: "남자부"를 선택하면 남자의 **모든** 하위
  층위 기록이 합쳐진 풀에서 시즌베스트를 계산한다. 고등학생이 일반부
  선수보다 빠르면 남자부 1위는 그 고등학생이다.
- **일반부 = 실업부(실업팀+무소속)를 지칭하는 하위 층위**다. 성별 롤업과
  절대 혼동하지 않는다. 원천 라벨 "남자일반부"만 여기에 매핑한다.
- **원천 라벨이 "남자부"뿐인 행**(대회가 층위를 구분하지 않은 경우)은
  `gender=남자, level=층위 미상`으로 둔다. **일반부로 강제 흡수 금지** —
  실업 대회가 많아 보여도 데이터가 말하지 않는 것을 단정하지 않는다
  (신뢰 원칙). 이 행들은 남자부(롤업)에는 포함되고, 일반부 필터에는
  나오지 않는다.
- U20/U18은 고등부와 겹치지만 **연령 기준과 학제 기준은 다른 축**이므로
  병합하지 않고 별도 층위로 유지한다.
- 학년별 라벨("남자중학교 2학년부")은 `level=중학부`로 올리되 원본 라벨은
  `divisionDetail`로 보존해 상세 화면에서 표시한다.
- 마스터즈 M35~W85는 `level=마스터즈`로 올리고 연령대(M45 등)는
  divisionDetail 배지로.
- `MA`/`WA` → 남자/여자 마스터즈. `통합부`/빈 라벨 → 성별 미상 + 층위 미상.

### 2.2 데이터 모델

각 record에 기존 `divisionKey/divisionLabel`을 대체·보강:

```js
{
  gender: 'men' | 'women' | 'mixed' | 'unknown',
  divisionLevel: 'general' | 'university' | 'high' | 'middle'
               | 'elementary' | 'u20' | 'u18' | 'masters' | 'unspecified',
  divisionKey: `${gender}-${divisionLevel}`,   // 통일된 단일 키 체계
  divisionLabel: '남자 일반부',                  // 표준 표시명
  divisionDetail: '남자중학교 2학년부' | 'M45' | null,  // 원본 보존
  rawDivision: '<원천 라벨 그대로>',             // 감사 추적용
}
```

- **TOP100 경로(`normalizeCandidateTopEvent`)와 data/results 경로
  (`normalizeEvent`)가 동일한 매핑 함수를 사용**해야 한다. 새 모듈
  `card-studio/services/divisionHierarchyService.js`로 분리하고 양쪽에서
  import. `kaaf-kind-*` 키 생성 코드는 제거.
- 매핑 테이블은 데이터에서 확인된 56종 라벨 + TOP100 category 14종을
  전부 명시적으로 커버하고, 미지 라벨은 `unspecified`로 흘리되
  경고 카운트를 summary에 남긴다 (조용히 삼키지 말 것).

### 2.3 시즌베스트 계산 규칙

- 시즌 테이블 버킷 키 = `season|eventKey|divisionKey` (통일 키 기준).
- **성별 롤업 버킷은 하위 층위 버킷의 합집합에서 계산**한다
  (men-general + men-high + … + men-unspecified → 남자부 버킷).
  구현은 롤업용 가상 버킷을 조회 시 합성하든, 인덱스 빌드 시
  이중 등록하든 자유 — 단 동일 선수 중복은 최고기록 1건으로.
- 같은 선수가 여러 층위에 기록이 있으면(전학·진학 연도) 각 층위
  버킷에는 각각, 롤업 버킷에는 최고 1건만.

### 2.4 UI 스펙 (확정)

- 부문 필터를 단일 드롭다운(75개)에서 **2단 선택**으로 교체:
  1. 성별 세그먼트: `남자 | 여자 | 혼성` (기본 남자)
  2. 층위 셀렉트: `전체(부 통합) | 일반부 | 고등부 | 대학부 | 중학부 |
     초등부 | U20 | U18 | 마스터즈 | 층위 미상`
     — **이 순서 고정** (사장님 확정: 일반 → 고등 → 대학 → 중학 순.
     가나다·데이터량 순 정렬 금지).
- 기본 화면 = `남자 × 전체(부 통합)` — 성별 롤업이 첫 화면.
- 표 행에는 층위 배지 표시 (롤업 화면에서 어느 부 기록인지 보이게):
  `일반` `고교` `대학` `중학` 등 소형 배지, divisionDetail 있으면
  툴팁/보조 텍스트로.
- '층위 미상' 필터는 데이터가 있을 때만 노출.
- 표시명 표준화: `남자고등학교부` → `남자 고등부`, `남자일반부` →
  `남자 일반부` 등. 상세 화면에는 rawDivision 병기 가능.

## 3. 계약 테스트 (`backend/tests/division-hierarchy.test.js` 신설)

1. 부문 필터 목록에 **동일 라벨 중복 키 0건**.
2. `kaaf-kind-` 접두 divisionKey가 인덱스에 존재하지 않음.
3. 층위 순서 고정: 일반부 → 고등부 → 대학부 → 중학부 → 초등부 →
   U20 → U18 → 마스터즈 → 층위 미상.
4. 롤업 규칙: 남자 고등부 선수 기록이 일반부 선수보다 좋은 픽스처에서
   `남자 × 전체` 시즌 1위 = 그 고등부 선수.
5. 일반부 필터에 '층위 미상'(원천 "남자부") 행이 섞이지 않음.
6. 학년별 라벨이 중학부로 롤업되고 divisionDetail 보존.
7. 동일 선수 롤업 버킷 중복 없음 (층위별 버킷에는 각각 존재 허용).
8. 미지 라벨 경고 카운터가 summary에 노출.
9. 기존 `manual-top-records-ingest.test.js`의 카운트 계약은 유지하되
   divisionKey 관련 단언은 새 키 체계로 갱신 (총량 24,630/16,885/7,745,
   승격 23,861/769, dedup 7,321은 변하면 안 됨 — 부문 키 변경은
   dedup 키(name|eventKey|date|record)와 무관하므로 delta 0이어야 정상.
   0이 아니면 중단하고 질문).
