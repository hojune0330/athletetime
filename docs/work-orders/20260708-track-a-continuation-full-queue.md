## A-2: 2016 전국대구실내육상경기대회 보류분 재파싱 (729행)

PR #35에서 `UNSAFE_EVENT_LABELS`(예: `60m 4`, `성명/소속/기록` 헤더 오염)로
보류한 1개 워크북을 전용 파서 보강으로 회수한다.

- 오염 패턴 분석: 종목 셀에 붙은 조/차수 접미(`60m 4` → `60m` + 조 정보),
  헤더 행이 데이터로 흘러든 케이스를 분류하고 각각 규칙化.
- 실내 대회 특성 주의: 60m/60mH/실내 트랙 종목은 야외 100m 등과
  **eventKey가 분리**되어야 한다 (기존 canonicalEventKey에 60m 계열이
  없으면 추가 — `60m`, `60m-hurdles`). 실내/야외 구분 플래그(`indoor: true`)를
  결과 JSON events에 남길 것.
- 회수 불가능한 행은 무리하지 말고 사유별 카운트와 함께 잔여 보류
  (`still-held.jsonl` + 사유). 목표는 100% 회수가 아니라 **안전한 최대 회수**.
- DoD: 스팟체크 2종목(원본 대조), 회수/잔여 카운트 보고, 테스트·빌드 그린.

## A-3: .xls 83파일 변환 트랙

레거시 바이너리 .xls(BIFF)의 순수 JS 파싱 경로를 확보한다.

1. **1차 시도**: 의존성 추가 없이 자체 BIFF8 리더가 과하면,
   잘 알려진 경량 라이브러리 1개(`xlsx`/SheetJS community edition 등)
   도입을 **PR 코멘트로 먼저 제안·승인받고** 진행 (의존성 정책 확인 필요).
2. 변환 성공 파일은 기존 xlsx 파이프라인(normalize → candidates →
   promotion)에 그대로 태운다. 새 파이프라인을 만들지 말 것.
3. 변환 실패 파일은 `blocked` + 사유 (암호화/손상/비표준). OCR·수기 입력은
   범위 밖.
4. 83개 중 연도 분포를 먼저 리포트하고, A-4~A-6 연도 묶음에 합류시킨다.
- DoD: 변환 성공률 리포트, 성공분 후보 큐 편입, 실패 목록·사유.

## A-4 ~ A-6: 연도 묶음 확장 (2012–2014 → 2009–2011 → 2005–2008)

각 묶음마다 동일 패턴:

1. 해당 연도 원본(xlsx + A-3에서 변환된 xls)을 normalize → candidates
   생성 (증거 큐 PR — 선택적으로 promotion과 합쳐도 됨, 단 PR 크기가
   과하면 분리).
2. 안전분 promotion → `data/results/<year>.json` + index.json 갱신.
3. 보류분은 사유별 held 처리 (A-2와 같은 방식).
4. **부문 키는 트랙 I의 divisionHierarchyService 기준** — kaaf-kind류
   임시 키 생성 금지.
5. 커버리지 카피·about-data·coverage-matrix를 같은 PR에서 갱신 (트랙 G).
6. year-checklist.csv 해당 연도 상태 전진.
7. PR 본문 보고 항목 (고정 서식):
   - 후보/승격/보류 행 수 (연도별)
   - `manualTopRecordStats.skippedDuplicates` delta — **2005–2017 확장부터는
     TOP100(2005–2017 커버)과 실제로 겹치기 시작하므로 delta > 0이
     정상이다.** delta와 대표 중복 예시 2건을 명시할 것.
     (참고: #35에서 delta 0이었던 것은 승격분이 TOP100 키와 미중첩이었기
     때문 — 앞으로도 0이면 그 사유를 설명해야 한다.)
   - 스팟체크 2–3개 대회 (원본 대조, 유명 선수 우선)
   - 안전 스캔 (forbiddenHits, git ls-files originals = 0)
   - 테스트/빌드 그린 + 프리뷰 URL
