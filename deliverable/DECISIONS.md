## D-001 · 메인 타깃 확정 (2026.04 초)

**결정**: 엘리트 선수 + 학생 선수 + 육상 팬/관계자

**대안으로 고려한 것**:
- 아마추어 러너/마라토너 → ❌ 제외
- 코치/지도자 → 부차적

**근거**: 호준님이 본인이 코치이면서, 사이트는 "육상 경기" 중심의 데이터·커뮤니티가 핵심. 건강 러닝 시장은 이미 다른 앱들이 잡고 있음.

**영향**: 디자인 톤이 "마라톤 동호회" 느낌이 아니라 "스포츠 매거진" 톤으로.

- 실제 코드는 Indigo `#6366f1` 메인 + Orange `#f97316` 액센트

## D-005 · 시즌 모드 도입 (2026.04 후반)

**결정**: 사이트가 시기에 따라 다른 얼굴
- **시즌 모드**: 대회 진행 중 → 라이브 대회가 헤로
- **비시즌 모드**: 대회 없을 때 → 검색바가 헤로

**근거**: 호준님 — "지금은 시즌이라 대회에 맞춰서. 사람들이 가장 많이 하는 반응과 행동에 맞춰 이 채널을 개선하거나 유지할 거거든."

**구현 (genspark 시안)**:
- `screens-kr/01_landing.html` (시즌 모드, 새로 작성)
- `screens-kr/01_landing_offseason.html` (기존 비시즌 보존)
- `screens-kr/07_competitions.html` (대회 목록 신규)
- `screens-kr/08_competition_detail.html` (대회 상세 라이브 대시보드 신규)

**커뮤니티 처리**:
- 시즌엔 대회 컨텍스트(전국체전 후기, 라이브 응원)가 위로
- 비시즌엔 일반 토픽(훈련일지, 부상복귀)이 위로

**상태**: 시안. 실제 코드 적용은 핸드오프 문서 단계에서.

## D-007 · 핸드오프 문서로 방향 전환 (2026.04 후반)

**결정**: 호준님이 만든 **실제 사이트 코드**가 이미 있음. 새로 디자인하지 말고, **그 코드 위에 적용할 디자인 지침 문서**를 만들기.

**근거**: 호준님 — "개발자가 너의 훌륭한 디자인을 따를 수 있는 지침을 만드는 일을 하자."

**핸드오프 문서 결정**:
- 기술 스택: React/Next.js (호준님 답변)
- 형태: 단일 HTML 인터랙티브 가이드 + 머지 가능한 마크다운
- 범위: Full (토큰 + 컴포넌트 + 페이지 + 데이터 타입 + 상태)
- 코드: 실제 복붙 가능한 JSX + Tailwind className
- 톤: 솔직하게, 시니어/주니어 모두 OK

## D-008 · 진짜 출발점 발견 (2026.04 후반, 가장 최근)

**결정**: GitHub 저장소 두 개를 발견하고 분석. 핸드오프는 `2026-first-item` 기준.

**과정**:
1. 호준님이 `https://github.com/hojune0330/2026-first-item` 공유
2. 첫 시도 GitHub Tree API 404 (기본 브랜치가 `main`이 아니라 `genspark_ai_developer`라서)
3. `https://api.github.com/repos/...` 직접 호출로 `default_branch` 발견
4. `https://github.com/hojune0330/2026-first-item/tree/genspark_ai_developer` URL로 정상 접근

**발견된 사실**:
- 호준님 톤은 "한글 심플 미니멀"인데 **실제 코드는 Indigo + Orange + 그라데이션** 사용 중
- `tailwind.config.js`에 이미 풍부한 컬러 시스템 (event 종목별 색상까지) 정의됨
- `index.css`에 이미 풍부한 컴포넌트 클래스 정의됨
- `frontend/src/components/trending/`에 4개 트렌딩 컴포넌트 이미 구현됨

**저장소 관계**:
- `athletetime` (메인) ← `2026-first-item` (실험·작업) 머지

**18개 핵심 파일 import 완료** (project root에 그대로 있음):
```
DESIGN_GUIDE.md, PROFILE_CARD_UX_PLAN.md, README.md
frontend/tailwind.config.js, src/index.css, src/App.tsx, src/App.css
frontend/package.json, src/config/constants.ts
src/components/trending/* (4개)
src/components/common/* (3개)
src/api/competitions.ts, trending.ts
```

**영향**: 이전 시안의 컬러 결정(`#FF3B30` 등)을 폐기. 실제 코드의 토큰을 따름.

## D-009 · 백업 시스템 구축 (2026.04 후반)

**결정**: 매 산출물을 standalone HTML로 만들어서 풀 URL 다운로드 링크 제공

**근거**: 이전 세션이 길어져서 끊김 → 작업물 유실 위기 (이 프로젝트 자체가 그 복구 작업의 결과물)

**구현**:
- `BACKUP.html` (백업 안내 + 원본 세션 복원 도구)
- `super_inline_html` 도구로 모든 산출물의 standalone 버전 생성 (`downloads/` 폴더)
- 파일명 규칙: `AthleTime_{산출물}_v{버전}_standalone.html`

**다운로드 URL 패턴**:
```
https://www.genspark.ai/api/designer2/serve/<프로젝트ID>/downloads/<파일명>
```
